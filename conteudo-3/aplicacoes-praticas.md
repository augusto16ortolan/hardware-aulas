# Aplicações práticas

Agora que você já comparou componentes e características, vamos traduzir isso para situações do mundo real.

## Onde o PC costuma ser mais usado

Em geral, PCs são a escolha mais comum quando o foco é:

- desenvolvimento de software com **IDEs robustas**;
- jogos;
- edição de vídeo;
- design gráfico;
- modelagem 3D;
- ciência de dados (em nível introdutório, por demandar mais recursos);
- virtualização;
- servidores de maior desempenho (dependendo do cenário);
- uso acadêmico e profissional em geral.

## Onde o Raspberry Pi costuma ser mais usado

Em geral, Raspberry Pi aparece muito quando o foco é:

- ensino de programação;
- ensino de eletrônica;
- prototipagem de IoT;
- automação residencial;
- robótica;
- servidores de baixo custo (simples);
- media center;
- monitoramento com sensores;
- projetos de hobby e pesquisa.

## Tabela de cenários

| Cenário | Melhor escolha | Justificativa |
|---|---|---|
| Jogar jogos pesados | PC | Precisa de GPU e desempenho gráfico maior |
| Rodar uma IDE com vários serviços locais | PC | Mais RAM/CPU e melhor multitarefa |
| Controlar sensores de temperatura | Raspberry Pi | GPIO e baixo consumo; ideal para projetos físicos |
| Criar um servidor web simples | Raspberry Pi | Baixo custo e bom para ficar ligado 24/7 |
| Fazer edição de vídeo | PC | Maior CPU/GPU, mais RAM e armazenamento rápido |
| Ensinar programação e eletrônica | Raspberry Pi | Integração com sensores/LEDs e foco educacional |
| Montar um projeto de automação residencial | Raspberry Pi | GPIO, baixo consumo e integração com módulos |
| Rodar máquinas virtuais | PC | Exige RAM/CPU e recursos de virtualização |
| Criar um media center barato | Raspberry Pi | Bom custo-benefício para reprodução de mídia |

## Dica de sala de aula

Sempre pergunte primeiro: “Qual é o objetivo do projeto?”

A escolha fica muito mais fácil quando você tem clareza do que precisa: desempenho gráfico, integração com sensores, consumo, custo, compatibilidade, etc.
